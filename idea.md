# GTO Poker Trainer — 기획 문서

> GTO 이론 기반으로 포커를 실전으로 배울 수 있는 웹사이트.
> 사용자는 GTO 봇과 직접 게임을 하고, 핸드 종료 후 chess.com 스타일로 복기하며 배운다.

---

## 1. 핵심 컨셉

- **게임 종류**: No-Limit Texas Hold'em (NLH)
- **테이블 형식**: 6-max (GTO 봇 5명 + 사용자 1명)
- **봇 특성**:
  - 사전 계산된 GTO 전략 테이블(룩업 방식)에 따라 플레이
  - 블러핑 없음 (순수 확률 기반 최적 전략만 사용)
  - 추후 블러핑 요소 추가 예정 (v2+)
- **타겟 유저**: 포커 초보부터 고수까지
  - 초보: GTO 기반 정석 플레이를 실전으로 학습
  - 고수: 완전 최적화된 봇 상대로 실력 점검/복습

---

## 2. 핵심 기능

### 2-1. 포커 게임 (Practice Game)

- 6-max 테이블에서 GTO 봇 5명과 대전
- 일반적인 NLH 게임 플로우:  
  프리플랍 → 플랍 → 턴 → 리버 → 쇼다운
- 봇은 각 스트릿(street)에서 GTO 전략에 따라 액션 (폴드/콜/레이즈)
- 사용자는 자유롭게 액션 선택
- 게임은 연속 핸드로 진행 (사용자가 원할 때 종료 가능)

### 2-2. 핸드 복기 (Hand Review) — chess.com 스타일

- **단위**: 한 핸드(라운드)마다 복기
- **점수 시스템**: 사용자의 각 액션을 GTO 최적 전략과 비교하여 등급 부여
  - 🟢 **최선의 수 (Best)** — GTO 최적 액션과 일치
  - 🔵 **좋은 수 (Good)** — 최적은 아니지만 수용 가능한 범위
  - 🟡 **실수 (Mistake)** — 명확한 EV 손실
  - 🔴 **블런더 (Blunder)** — 심각한 EV 손실
- **전체 요약**: 핸드 내 각 등급 비율 표시 (예: 최선의 수 60%, 좋은 수 20%, 실수 15%, 블런더 5%)
- **코치 설명**: 각 액션에 대해 왜 그것이 최적/비최적인지 설명
  - 설명 방식: 조사 필요 (비싼 LLM 사용 최소화 방향)
  - 가능한 접근법: 사전 정의된 템플릿 기반 설명, 소형 LLM, 또는 룰 기반 설명 생성 등

## 5. 조사 필요 항목

> 아래 항목들은 개발 전 리서치가 필요한 사항들이다.

### 5-1. GTO 전략 데이터 소싱

- 사전 계산된 GTO 솔루션 데이터를 어디서 확보할 것인가?
- 오픈소스 GTO 데이터 존재 여부 조사
- 프리플랍 차트 (이미 완전히 풀린 솔루션 존재)
- 포스트플랍 주요 상황 데이터
- 데이터 포맷과 저장 방식 설계

### 5-2. 복기 코치 구현 방식

- 비싼 LLM(GPT-4 등) 사용을 최소화하는 방향
- 고려할 수 있는 방식들:
  - 룰 기반 템플릿 설명 (가장 저렴)
  - 소형 오픈소스 LLM 활용 (로컬 or 저렴한 API)
  - 하이브리드: 기본은 템플릿, 상세 설명 요청 시만 LLM 호출
- 비용 대비 설명 품질 트레이드오프 분석

### 5-3. 데이터베이스 선택

- 저장해야 할 데이터: 사용자 정보, 게임 히스토리, 복기 기록, 하루 무료 복기 횟수 추적
- 후보: Supabase(PostgreSQL), Firebase, PlanetScale 등

### 5-4. 결제 시스템

- 구독제 vs 건당 결제 vs 크레딧 방식
- Stripe, Toss Payments 등 결제 수단

### 5-5. 카드게임 프론트엔드 UI 라이브러리

- 포커 테이블, 카드 렌더링, 칩 애니메이션 등을 직접 구현하지 않고 **GitHub 오픈소스 활용**
- 조사 대상:
  - React 기반 포커/카드게임 UI 컴포넌트 라이브러리
  - 카드 덱 렌더링 라이브러리 (SVG 카드 에셋 포함)
  - 포커 테이블 레이아웃 및 애니메이션 라이브러리
  - 포커 핸드 평가(hand evaluator) 라이브러리
- 선정 기준: 스타 수, 유지보수 상태, React/JS 호환성, 라이선스
- 필요 시 fork하여 커스터마이징

## 6. MVP 범위

### MVP에 포함

- [x] 6-max NLH 게임 (봇 5 + 유저 1)
- [x] GTO 전략 테이블 기반 봇 (블러핑 없음)
- [x] 핸드 복기 시스템 (점수화 + 코치 설명)
- [x] Google 로그인
- [x] 하루 1회 무료 복기 제한

### MVP 이후 (v2+)

- [ ] 봇 블러핑 추가
- [ ] 결제 시스템 연동
- [ ] 게임 히스토리 저장 및 통계 대시보드
- [ ] 난이도 조절 (GTO 준수율 조절)
- [ ] 모바일 반응형 최적화
